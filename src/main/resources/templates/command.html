Git Commands Guide

A comprehensive guide to Git commands with detailed explanations.

1. Initialize a Git Repository

This command initializes an empty Git repository in the current directory.

Once you run this command, the directory becomes a Git repository and a .git folder is created. This folder contains all the version control data.

git init

2. Configure Git User

Set up your Git username and email address globally. This information will be used in your commit history.

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

3. Check Git Configuration

This command displays the current Git configuration settings, such as user name, email, and other preferences.

git config --list

4. Add Files to Staging

Stage the changes made to files before committing them. This tells Git that these files should be included in the next commit.

git add filename

5. Commit Changes

Commit the staged changes to the local repository with a descriptive message about the changes made.

git commit -m "Commit message"
