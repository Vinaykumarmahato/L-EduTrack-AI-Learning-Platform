Introduction to JDBC

JDBC (Java Database Connectivity) is a standard Java API that enables Java applications to connect and interact with databases. It provides methods for querying and updating data in a database using SQL. JDBC is widely used for database operations in Java applications, including web and desktop apps.

What is JDBC?

JDBC is an interface between Java applications and databases. It allows developers to perform basic database operations like creating, reading, updating, and deleting records (CRUD) using SQL queries from within a Java program.

Why Use JDBC?

  • Database independence: Use any database with just a different driver.
  • Supports all SQL operations (DDL, DML, DCL).
  • Can execute stored procedures and manage transactions.
  • Lightweight and included with Java SE.

Key Components of JDBC

  1. DriverManager: Loads JDBC drivers and manages database connections.
  2. Connection: Represents a session with the database.
  3. Statement: Executes SQL queries.
  4. ResultSet: Holds results retrieved from the database.

Basic JDBC Workflow

The steps to use JDBC in an application:

  1. Load the database driver class.
  2. Establish a connection using DriverManager.getConnection().
  3. Create a Statement or PreparedStatement.
  4. Execute SQL commands like executeQuery() or executeUpdate().
  5. Process the ResultSet if needed.
  6. Close the connection to free resources.

Example


Class.forName("com.mysql.cj.jdbc.Driver");
Connection con = DriverManager.getConnection(
    "jdbc:mysql://localhost:3306/mydb", "user", "password");
Statement stmt = con.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM users");
while(rs.next()) {
    System.out.println(rs.getString("username"));
}
con.close();

Advantages

  • Direct control over SQL execution.
  • Supports all major relational databases.
  • Good for small to mid-level database applications.

Limitations

  • Verbose code with lots of boilerplate.
  • No automatic object-relational mapping.
  • Error-prone if connection/resources are not managed properly.

Conclusion

JDBC provides the foundation for interacting with databases in Java. Whether you're developing a simple student record system or a complex enterprise platform, JDBC knowledge is essential to understanding Java's backend data handling capabilities.
